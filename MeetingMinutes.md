## End-of-Quarter Retrospective

### What Went Well

Overall, our team worked together well throughout the quarter. We were able to split up tasks, help each other when someone got stuck, and finish our sprint work on time. Communication improved as the quarter went on, and by the final few sprints everyone had a good understanding of what needed to be done. We are proud that we were able to build and deploy a complete application that included authentication, reviews, ratings, and integration with another team's API.

### What We Would Do Differently

If we were starting over, we would spend more time planning our project structure earlier in the quarter. As the project grew, there were times when we had to go back and reorganize code or clean up things that were rushed in earlier sprints. We would also test deployment changes sooner instead of waiting until the end of a sprint, since many of our issues came from differences between local development and production.

### Working With Partner APIs

Working with another team's API was one of the most interesting parts of the project. It showed us how important good documentation and communication are. Sometimes there were issues with endpoints, deployment, or data formats, and we had to work together to figure them out. It also gave us a better understanding of what other developers experience when using an API that we created ourselves.

### Working With AI Coding Tools

AI tools were helpful throughout the project, especially for debugging, explaining errors, and helping us learn new concepts more quickly. At the same time, we learned that AI is not always correct and that we still needed to understand the code before using it. There were several situations where AI-generated solutions did not work as expected or needed changes to fit our project. One of the biggest lessons from the quarter was learning how to use AI as a tool while still thinking critically about the code ourselves.

### Final Thoughts

This project gave us experience working on a larger application than we had built before. We learned a lot about teamwork, Git workflows, deployments, APIs, and front-end development. Even though there were challenges throughout the quarter, it was rewarding to see the final application come together and be something we could actually use and demonstrate.
